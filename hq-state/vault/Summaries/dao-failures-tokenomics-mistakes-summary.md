# Summary of dao-failures-tokenomics-mistakes

- DAO failures often come from **rigid tokenomics**: fixed initial distribution and parameters can’t adapt to changing community needs, so governance token mechanics should allow parameter updates via proposal.
- **Voting power concentration** from poor initial token distribution creates plutocracy; early investors/team holding too much supply can capture governance, and mechanisms like dissolve delay/age bonuses may compound this in [[neuron-staking-voting-power-mechanics]].
- **The DAO (2016)** showed that trustless governance must address adversarial scenarios, edge cases, and smart contract vulnerabilities; governance design needs expanded theories beyond traditional corporate models.
- Key DAO design decisions include voting power allocation, how votes translate to outcomes, quorum/conviction rules, and social choice theory applied to tokenomics.
- Treasury failures stem from unclear spend governance, poor adaptability in treasury allocation, and weak spending limits; see [[sns-treasury-and-initial-token-distribution]] for allocation best practices.
- For SNS init file design: enable meta-governance, prevent early whale concentration with vesting/lockups/caps, add treasury budgets, and consider reputation/commitment signals like dissolve delay alongside raw token holdings.
