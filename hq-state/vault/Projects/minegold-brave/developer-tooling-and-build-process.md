# Developer Tooling and Build Process

This project uses a `pnpm` workspace to manage a monorepo structure, as defined in `pnpm-workspace.yaml`. This allows for separate handling of frontend and backend dependencies and build processes while keeping them in a single repository. The core tooling revolves around `pnpm` for the frontend and `mops` for the Motoko-based backend.

## Monorepo Scripts

The root `package.json` defines several scripts that run commands recursively across all packages in the workspace (e.g., `pnpm -r --if-present run build`).

```json
{
  "scripts": {
    "build": "pnpm -r --if-present run build",
    "typecheck": "pnpm -r --if-present run typecheck",
    "check": "pnpm -r --if-present run check",
    "fix": "pnpm -r --if-present run fix",
    "bindgen": "caffeine-bindgen --did-file ./src/backend/dist/backend.did --out-dir ./src/frontend/src --actor-interface-file --force"
  }
}
```

## Verified Workflow

The `AGENTS.md` file outlines the verified, step-by-step commands for setting up and building the project.

### Frontend (`src/frontend/`)

The frontend uses `pnpm` for its package management and scripts.

- **Install dependencies**: `pnpm install --prefer-offline`
- **Typecheck**: `pnpm typecheck`
- **Lint & Fix**: `pnpm fix`
- **Build**: `pnpm build`

### Backend (`src/backend/`)

The backend uses `mops`, the package manager for Motoko.

- **Install dependencies**: `mops install`
- **Typecheck**: `mops check --fix`
- **Build**: `mops build`

## Frontend-Backend Integration

A critical step for connecting the two parts of the application is generating type-safe bindings. This is handled by the `caffeine-bindgen` tool, run via the root `bindgen` script.

```bash
pnpm bindgen
```

This command reads the Candid interface file (`.did`) generated by the backend build and creates corresponding TypeScript files in the frontend's source directory. This ensures that any calls from the [[frontend-and-backend-workflows|frontend]] to the [[backend-core-implementation|backend]] are type-safe and match the deployed canister's public interface.
