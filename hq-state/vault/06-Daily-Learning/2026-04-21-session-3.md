# 2026-04-21 — Session 3: SNS configs and setup

Third research session. The focus today is the **Service Nervous System (SNS)** — how ICP packages the NNS governance pattern so an individual dapp can be handed off to a DAO.

## Pivot

Session 2's closing log proposed three priorities for Session 3:

1. `[[local-replica]]` + `[[mainnet-deploy]]` written as a pair
2. `[[cycles-wallet]]`
3. Promoting `[[dfx]]` to `stable`

The user **explicitly redirected this session to SNS** instead. The three Session-2 priorities are **deferred** and are carried forward as candidates for a future session — they remain accurate open gaps in the vault's Deployment branch.

## New notes

Five atomic concept notes — all `status: draft`, all qualitative, all linked off a new **Governance** MOC section:

- [[sns]] — umbrella note. What an SNS is, relationship to the [[nns|NNS]], and the three-stage decentralization lifecycle (dev-controlled → NNS-controlled transient → DAO-controlled).
- [[sns-canisters]] — the canister cluster: Root, Governance, Ledger (ICRC-1/2), Index, Swap, Archive. Responsibilities and control graph (Governance controls Root; Root controls everything else and the dapp canister).
- [[sns-init-config]] — the `sns_init.yaml` surface. Describes the major *sections* (Institutional / Token / Initial Distribution / Governance Parameters / Neurons / Swap / Neurons' Fund) qualitatively — **does not transcribe field names**, since those drift; the SNS CLI template is cited as the authoritative schema.
- [[sns-launch]] — launch lifecycle: dev-side preconditions (NNS Root co-controller, validated config, fallback controllers), the single `CreateServiceNervousSystem` NNS proposal, SNS-W deployment, the timed pre-swap window, the swap itself, and the success/failure branches.
- [[sns-neurons-and-voting]] — neuron creation (swap basket at launch; post-launch stake-to-claim), dissolve delay, qualitative voting-power drivers, following (topic-scoped), and common proposal categories (canister upgrades, parameter changes, treasury actions, motions, custom types).

And one tutorial:

- [[sns-init-yaml-walkthrough]] — 30-minute walk-through of `sns init-config-file new` → `validate` → optional `deploy-test-flight` / `--network local` round trip. Explicit source-of-truth caveat at the top: the template emitted by the current CLI is authoritative; the tutorial describes section *shapes*, not field names.
